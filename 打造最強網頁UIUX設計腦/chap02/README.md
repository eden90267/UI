# chap02：將網頁設計當成一種溝通

## 傳遞訊息

### 一眼就看懂是件重要的事

設計最重要的就是「一眼就看懂」這件事。智慧型手機的網頁設計/應用程式設計，更需要「讓使用者立刻掌握內容」。

設計目的在於整理資訊與傳遞資訊。

這也包含盡可能**讓人們減少動腦思考/計算**這件事。

訊息圖表，以視覺設計呈現資訊、資料與知識的圖片。

無形的資料轉化視覺設計，並不是為了要求使用者解讀圖片的意思，而是要讓使用者一眼就能看懂要傳遞的意思。

有時必須視情況思考使用者當下最需要哪些資訊。

### 思考「使用者現在最需要的資訊是什麼」這個問題

使用者需要哪些資訊的問題，只要想想看使用者在**什麼時候**、**什麼狀況**下使用，就能找到正確答案。

設計的最佳解答會隨著該提供哪些服務，以及將重點放在哪些服務上而改變。

**想要「一眼就看懂」，就必須掌握服務的全貌**

進一步考慮使用者會如何使用系統，傾聽使用者的意見，然後再決定哪些是必要的元素。

- 使用者會在何種情況下使用
- 最重要的資訊是什麼

在此之前，請先掌握服務的全貌。

### 面對「為什麼會編排出這種版面」的詢問時，也能清楚回答的根據

「為什麼會編排出這種版面？」不得不只憑自己的感覺回答時，通常只回答「憑個人感覺設計」的這種不太可靠的答案。

假設「想讓某部分變得更搶眼」，結果只憑毫無根據的創意與感覺賦予版面變化，有時反會另使用者產生誤解。

以下介紹一些法則，作為決定設計時的指標。

#### 格式塔(Gestalt)

```
●●●    ●●●    ●●●    ●●●
●●●    ●●●    ●●●    ●●●
●●      ●●    ●●●    ●●
●●      ●●    ●       ●●
```

每個小點不具意義，但集合起來就會產生意義的這個現象稱為「格式塔」。

格式塔的理論不只包含視覺集合體，還包含聽覺集合體。「Do」這個音階，應該聽不出哪首曲子，但「Do Re Mi Do Mi Do Mi」就聽得出是真善美這首歌。

人類本來就擁有**搜尋集合體資訊的特性(本能)**。

人類不擅長處理過於瑣碎的資訊，一旦資訊量暴增，不僅無法掌握，更無法清楚說明。

但，從遠處觀察，**並從事物全貌(格式塔)找出特徵**，**這些事物就很容易進入人類意識裡**。

#### 距離與關係

人類的視覺會將具有一定法則(模式)的編排方式視為「群組」。

- 依位置組成群組的情況
- 以列表元素組成的格式塔
    
    ```
    1.
    ● A
    ● B
    ● C
    ● D
    
    2.
      A
    ● B
    ● C
    ● D
    
    3.
    ● A
    ● B
    ● C
    
    ● D
    ```
    
    即便是相同層級的資訊，只要有**部分的資訊不符規則(模式)，就會被當成例外的資訊**。
    
    若想強調列表裡的其中一項資訊，建議不要只改變與其他列表資訊**共用的圖示**或**留白**的規則。

- 與照片或文章有關的格式塔法則

    在網頁設計裡，「說明這部分文章的是這張照片」的資訊關聯性必須簡化而清楚。
    
    將相關資訊以框線或背景色統整，版面更加穩定，也讓使用者更想瀏覽內容。
    
    奇特的照片配置方式與不符合常規的配色一多，**網頁的內容就會顯得很混亂，許多使用者也會疲於瀏覽，也無法集中精神閱讀內容**。

- 封閉空間

    尤其是行動應用程式裡較容易瀏覽的標籤列與導覽列設計，明確劃分出主要內容區塊才能提升使用者的理解度。

#### 類似、共通的模式

使用相同裝飾，可讓元素被當成能以相同方式操作的元素。

#### 容易因為製作者的企圖而被誤解的格狀系統版面

「因配置組成的群組」原理，會導致編排上的錯誤。

```

      主要視覺設計

特徵一 特徵二 特徵三 外部連結
```

「外部連結」是導致認知錯誤的範例。

## 與使用者建立溝通

### 紙本與網頁的兩大差異

#### 互動性(隨時對話的狀態)

網站有許多可解決使用者「想做更多~」的方法。

使用者選圖片放大，按Close回到原本畫面。點選Next還可瀏覽其他圖片。

但互動設計要明顯或提醒方式讓使用者知道。

#### 動態設計(只切換到需要的資訊)

搜尋資料的結果顯示之前，先顯示搜尋結果數量，也是一種使用動態網頁的方法。

### 以不動產搜尋網站為例，介紹兩項動態設計重點與使用者行為

#### 將焦點放在使用者行為的重要性

如果設計師能將焦點放在使用者行為，在製作物件搜尋的Wireframe時，就能清楚說出自己的意思。

能利用文字大小、配置方式與顏色，**提出最適合使用者使用的方案，也是網頁設計師要求的課題之一**。

#### 先顯示搜尋結果數量，若是資料太多就立刻篩選

先讓使用者了解搜尋結果可能會爆增的問題，應該可減輕使用者的負擔。

為此，設計師應該要將搜尋結果放在使用者立刻能看得到的位置，或是設定成搶眼的顏色。

很多網站在使用者輸入搜尋文字，就立刻向伺服器提出詢問Ajax技術。這代表設計師必須設計出能**先行預測使用者行動**的版型。

#### 為了連書籤都不熟悉，瀏覽能力不足的使用者進行設計

為讓不熟悉網路瀏覽流程的使用者，必須設計出更方便使用的書籤功能。也需要能輕鬆代替書籤物件的功能。

將資訊編排成不需要加上書籤就能閱讀的程度，可讓那些覺得書籤功能難用的的人覺得更加方便。

加上書籤後，其他區域明顯顯示"書籤一覽"的功能連結，並**閃亮**邊框提示。

設計師雖然設計了可在物件貼上書籤的功能，但是該如何解決使用時的「不滿」，也是設計師在設計之際必須思考的事情。

設計在喜歡的物件中貼上書籤的功能時，必須思考以下幾點：

1. 不註冊也能使用
2. 誰都能注意到標籤的存在

    - 貼上書籤的資訊，可從哪裡的按鈕開啟一覽表?
    - 按下哪裡的按鈕，可在頁面的資訊貼上書籤?

3. 不需要任何技巧就能貼上標籤
4. 可立刻撕下標籤
5. 可輕鬆將貼上標籤的物件傳給別人看


### 與使用者的溝通

當我們在思考該如何與使用者建立溝通管道時，或許會覺得直接問使用者「你現在在搜尋什麼?」的管理員功能是最理想的方法。

#### 使用者進入這個頁面，到底想了解什麼事情呢?

GA分析工具可分析使用者的行為模式(e.g. 在首頁裡最常被點選的按鈕是哪一個)，但是卻沒有辦法直接問使用者「你到底想要哪些資訊」這個問題。

其中一種方法是先站在使用者的立場，與其說是一直盯著網站，不如說是像在進行日常的對話一樣。

#### 思考符合時代潮流的訴求

大量消費的時代已經結束，使用者漸漸喜歡具有個性的商品。

消費者已放棄透過某些商品尋找夢想，而是希望商品能更實用。

溝通這方面的管道，在設計時應納入考慮。

#### 官方網站的情況

營造夢想的企業業務內容，太抽象，使用者難懂。換成日常與使用者對話來看，就會立刻明白原因。

#### 電子商務網站的情況

電子商務網站指在網站上購買商品，然後商品就會經由宅配送到家的所有服務。

僅將銷售的商品擺放在網站裡，可是無法宣傳商品的魅力。

為此，才需要大量使用照片與文章，藉此與使用者溝通。

最有效的方法，果然還是**對話**。

問自己「到底是什麼樣的圍裙?」、「這件圍裙與其他的圍裙有什麼不一樣」來自使用者的問題，能否回答?

仔細想想，到底使用者實際到店裡會問什麼樣的問題? 然後將它套用在網站商品文章與設計上。

#### 若無其事地促請顧客體驗

應用程式的宣傳短片。這樣促銷會得到意想不到的收穫。

#### 仔細一問，才發現許多使用者不了解自己的目的

提出圍裙使用方法。

- 在廚房穿
- 整理自家庭院時穿
- 當成咖啡館的制服穿
- 出門也可以穿

營造使用者的心理效果。

- 會被朋友或老公稱讚
- 會忘了自己身在廚房，在不知不覺之間廚藝進步
- 廚房與自己也會擁有明亮的笑容

這些會出現使用者**從網站的溝通誕生的恍然大悟**。

目的 -> 行為

「想要作這類的事情(目的，也是希望) -> 所以買圍裙」，可發現目的變成了希望。

只要有這類經驗，使用者就能在原本曖昧不明的目的加上附加價值，也能更愉悅進行購物。

為進一步讓使用者了解商品的魅力，必須重視下列幾點：

- 闡述商品的堅持之處
- 換成人與人之間面對面的對話模式，思考與使用者之間的對話
- 讓使用者想像擁有這項商品會有什麼好處

## 將溝通放進設計裡

### 為什麼紙本媒體的設計無法直接套用在網頁呢?

#### 網頁的內容比紙本媒體的內容還方便跳著閱讀的理由

「在報紙或雜誌的完美編排，到了網頁就是完蛋的編排」 - **使用者所能接收的資訊量有其極限**

網頁的顯示範圍不像紙本一樣固定，使用者無法一眼看出哪裡到哪裡是文章的一個段落。

網頁的內容之所以比較方便跳著閱讀，在於不容易掌握文章結尾處這個理由。

#### 跳脫模式與集中注意力

報紙或雜誌的B4閱讀範圍，怕有些人覺得排在格狀版面裡的文字有點無聊，因此，報紙或雜誌常故意不規則編排內容。

但網頁並非A4或B4固定區域大小的媒體，對於想在網頁跳著閱讀內容的使用者來說，這種不規則的編排反而看不出哪裡到哪裡是一個章節或段落。

對於習慣跳著閱讀的使用者來說，所謂容易閱讀的安全版型設計，就是盡可能遵守編排規則的版型。雖然單調，但越單調的設計越適合編排長篇的文章。

因此，若將紙本媒體的設計直接套用在網頁設計裡，會讓版面變得很凌亂，也會降低使用者的閱讀意願。

#### 網頁設計不一定非得單調

**了解企業的概念，在諸多戰略之中，大膽採用跳脫規則的網頁設計**或許也是一種創新。

不管是要設計單調的模式，或是要勇敢跳脫既有模式，請先從**內容的數量與質量**考慮。一切都得視情況而定。

### 思考瀏覽一覽表的使用者心理

#### 使用者照訪網站的過程

搜尋功能，要想想看使用者是抱持何種期待進入入口網站的。

開發人員很容易陷入「要顯示較多的店面資訊」的迷思，要注意的是，使用者在找到店面資訊之前，就已經「先有想解決的問題」了。

以寵物入口網站為例，搜尋功能不該只提供店名與地址，而要以「貓砂盆」這類**使用者可能會使用的關鍵字製作一覽表，否則使用者一定會拋棄這個寵物入口網站**。

#### 該如何觸動使用者的心

設計師該仔細思考「令使用者感興趣的元素」。若只是制式地植入「**詳細介紹請點選這裡」的按鈕是毫無意義的**。

以女性衣服為例，**充滿魅力的照片**、**強調特徵的文案**、**折扣**與**優惠的資訊**，才能真正讓使用者心有所感。

要讓使用者感興趣的重點有下列三點：

- 標題的寫法或文案的寫法
- 照片的品質與角度
- 次要文案應力求簡潔

讓忙碌的使用者先粗略地閱讀短篇文章，若看到喜歡的內容還可以迅速地點選。

### 製作扁平化設計的原則

#### 平面不一定是正確解答，一切得視個案而定

#### 從設計的歷史學習扁平化設計

轉盤式電話與扁平化的按鍵式電話，扁平化設計普及的因素：

- 有利於生產成本的因素
- 技術革新
- 不再需要旋轉轉盤

只要符合機械與人類的需求，該設計就能得以普及，這就是扁平化設計得以普及的重點。這與GUI也與相通之處。

介面的本質就是「重視人類與機械關係設計」。

#### 為什麼按鈕一按就會凹下去

物理的按鈕特性(鍵盤)套用在按了也不會凹下去的類型裝置上面(平板、智慧型手機)。

#### 考慮使用者的接受度

設計「可按壓的感覺」即可，e.g. IOS 7的簡單漸層色。

#### 連結不需做成按鈕，只要套用底線

最近的網頁設計以「方便使用」為主流，不再那麼強調「美觀」。

#### 扁平化的本質

很多人認為扁平化設計就是「外觀看起來為平面」的意思。但廣泛而言，應該想成「人與機械關係扁平化」的概念。

簡化必要的操作，可讓「人與機械的關係」變得圓融。

主角與配角關係變得更加清楚。影片畫面與GUI播放器需做搶眼方面的設計，誰為主誰為配。到底想傳遞給人的「內容」是什麼? 只有釐清這點，人們才能進入內容的世界裡。

過於豐富的設計，會讓擔任主角的內容失去地位。但簡單的設計，還是得注意使用者是習慣操作電腦，或是熟悉平板電腦、智慧型手機的操作。

### 企業更需要宣傳「能面對面的安心感」

「我拜託的不是公司而是你」。

顧客看企業，看的不是公司名稱而是人。

#### 「社長才應該是門面」

#### 增加支持者

網站刊登社長、店長或員工那不做作的姿態，就能讓顧客稍為安心一點。

顧客在購物時，內心都希望該公司或店家能親切與仔細地接待他們。只要能做到這點，就有機會增加支持者。

#### 在網站刊登照片

能看見臉，就會覺得那個人像是鄰居，也會覺得比較安心。

不管公司是大是小，在製作直接提供服務的公司網站時，為了「讓第一次上門的顧客能夠安心，一定要刊登照片讓顧客知道是誰在接待他們」

#### 移開視線再拍是重點

多數人都沒辦法看著鏡頭露出很自然的表情。

最好不要從正面拍，而是從斜側面拍過去，這樣可稍微緩和緊張的感覺。

#### 拍出毫不做作的照片

比起專業攝影技術，**這裡更重視的是社長開心聊天時按下快門**。

能夠傳遞「社長樂於接待客人」這點，比什麼都重要。
